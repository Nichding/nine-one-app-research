# Anti-Commodity Principles

## The danger

Nine One must not become interchangeable.

The easiest path is to become another app for booking, paying, listing courts, sending reminders, and showing profiles.

That path leads to commodity competition: more inventory, lower fees, more features, more noise, less meaning.

Nine One should choose a harder and better path: build the trust, memory, and improvement layer around padel.

## Principle 1: Do not worship inventory

Court inventory matters, but inventory is not the soul of the product.

A court without the right players is empty in a deeper sense.

Nine One should care less about how many courts are listed and more about how many good matches are completed.

| Commodity question | Nine One question |
| --- | --- |
| How many courts do we list? | How many players found the right match? |
| How many bookings happened? | How many people came back? |
| How many venues signed? | How many venue communities became stronger? |

## Principle 2: Quality beats volume

Bad liquidity is worse than low liquidity.

If players join mismatched, unreliable, or awkward games, they learn not to trust the system.

Nine One should prefer fewer matches that feel right over many matches that feel random.

## Principle 3: Trust is a product feature

Trust is not brand language. It is built through rules and memory.

| Trust signal | Why it matters |
| --- | --- |
| Reliability | People need to know others will show up. |
| Level confidence | People need to know the match will fit. |
| Clear cancellation rules | People need fairness before committing. |
| Coach validation | People trust human judgment when scores are uncertain. |
| Transparent changes | People need to understand why their profile changes. |

## Principle 4: Progress must feel personal, not mechanical

Players do not want to be processed by an algorithm.

They want to feel seen.

Nine One should make improvement specific, humane, and actionable. One clear next step is better than a page of statistics.

## Principle 5: The venue is a host, not supply

Many platforms treat venues as units of availability.

Nine One should treat venues as living places.

A good venue has character, regulars, coaches, rituals, and trust. The product should help that character become stronger, not flatten every club into the same marketplace tile.

## Principle 6: AI must earn its place

AI should not be decoration.

It should not produce confident nonsense.

It should not replace coaches before it has learned from them.

AI belongs where it can make the next match, lesson, or decision better.

| Good AI | Bad AI |
| --- | --- |
| Explains level confidence | Hides rating changes |
| Suggests a useful next action | Generates generic advice |
| Learns from coach labels | Pretends human expertise is obsolete |
| Improves match balance | Maximizes engagement without care |

## Principle 7: Do not build for endless attention

Nine One should not try to keep people in the app.

It should help people get out and play.

The best sign of success is not scrolling time. It is a completed match, a better group, a useful lesson, a return visit, a stronger club.

## Principle 8: Data must give back

If Nine One collects data, the user should feel the return.

| Data collected | Return owed |
| --- | --- |
| Attendance | Better reliability and fairer matching |
| Match feedback | Better future matches |
| Skill signals | Clearer level confidence |
| Improvement goals | Better coaching recommendations |
| Venue demand | Better programming and availability |

Data without returned value is extraction.

## Principle 9: Refuse shallow growth

Nine One should not grow in ways that weaken the game.

| Refuse | Reason |
| --- | --- |
| Empty market launches | They create dead experiences. |
| Unverified ratings at scale | They damage match trust. |
| Hidden fees | They corrode commitment. |
| Generic engagement mechanics | They distract from real play. |
| Replacing venue identity | It weakens the places that make sport meaningful. |

## Principle 10: Build memory

Commodity tools forget.

Nine One should remember.

It should remember who shows up, who plays well together, which matches felt balanced, which coaches helped, which venues became homes, and which next steps led to real improvement.

Memory is the opposite of commodity.

## The anti-commodity test

Before building anything, ask:

| Question | If the answer is no |
| --- | --- |
| Does this make matches better? | Do not build it yet. |
| Does this make players more trusted or more understood? | Do not build it yet. |
| Does this help venues host stronger communities? | Do not build it yet. |
| Does this help coaches guide real improvement? | Do not build it yet. |
| Does this create useful memory over time? | Do not build it yet. |
| Would users miss it if they left Nine One? | Do not build it yet. |
