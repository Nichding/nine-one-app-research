# Emotional Product Design

## Emotional design thesis

Nine One should make sport feel easier to enter, safer to repeat, and richer to remember.

The emotional state is:

**grounded confidence.**

Not hype. Not status. Not pressure. Not endless optimization.

Grounded confidence means:

| Feeling | Product cause |
| --- | --- |
| Welcomed | There is a clear way into a match. |
| Matched | The level and people feel right. |
| Trusted | Commitment and reliability are visible. |
| Clear | The rules and next step are obvious. |
| Capable | Progress is visible without shame. |
| Connected | Familiar players and venues return. |

## Emotional positioning in product behavior

| Product moment | Emotional design rule |
| --- | --- |
| Onboarding | Ask only what helps the user find the right match. |
| Match recommendation | Show why this match fits. |
| Joining | Make commitment feel serious but fair. |
| Before match | Reduce uncertainty. |
| After match | Turn effort into memory. |
| Recap | Give one useful truth, not a flood of stats. |
| Level update | Explain confidence gently. |
| Reliability | Treat attendance as respect, not punishment. |
| Coach prompt | Make help feel available, not corrective. |

## Words the product should earn

Nine One should eventually earn these words from users:

| Word | Meaning |
| --- | --- |
| "I trust it." | Matches feel reliable and fair. |
| "It knows my level." | Recommendations feel accurate. |
| "I know people there now." | Community is forming. |
| "I am getting better." | Progress feels real. |
| "This is my club." | Venue attachment is forming. |
| "I know what to do next." | Guidance is working. |

## Product tone

| Use | Avoid |
| --- | --- |
| Clear, calm language | Overexcited growth language |
| Human explanations | Algorithmic mystery |
| Match-specific guidance | Generic motivation |
| Respectful correction | Shame-based performance framing |
| Commitment language | Casual slot-shopping language |
| Place and people | Abstract engagement |

## Emotional attachment mechanisms

| Mechanism | How it works |
| --- | --- |
| Recognition | The product remembers who the player is becoming. |
| Reliability | Showing up becomes part of identity. |
| Familiarity | Repeated players and venues create belonging. |
| Progress | Recaps show change over time. |
| Guidance | The next step feels achievable. |
| Local pride | The venue becomes a meaningful place, not just a court. |

## What the product must never make users feel

| Feeling | What causes it |
| --- | --- |
| Exposed | Too much public scoring or ranking. |
| Confused | Hidden ratings, unclear fees, unclear rules. |
| Replaceable | Treating players as anonymous booking units. |
| Trapped | Lock-in without returned value. |
| Judged | Harsh performance framing or unexplained AI output. |
| Alone | Empty feeds or matches with no visible community. |

## Emotional design of data

Data should feel like memory, not surveillance.

| Data moment | Emotional framing |
| --- | --- |
| Attendance | "You are reliable." |
| Level confidence | "We are learning where you fit." |
| Balance rating | "Your future matches can be better." |
| Improvement tag | "Here is one thing to work on." |
| Coach validation | "A human has seen your game." |
| Venue history | "This place is becoming part of your rhythm." |

## V1 emotional hierarchy

| Priority | Emotion | Product requirement |
| --- | --- | --- |
| 1 | Trust | Clear players, rules, payment, attendance, reliability. |
| 2 | Fit | Level confidence and match recommendation. |
| 3 | Belonging | Home venue and familiar players. |
| 4 | Progress | Recap and next action. |
| 5 | Pride | Reliable-player identity and improvement over time. |
