# Player Retention Engine

## Retention thesis

Nine One should not retain players with notifications, streak tricks, or a generic feed.

It should retain players because:

**The next match is easier to find, safer to join, more likely to fit, and more meaningful after it ends.**

Retention comes from rhythm, trust, progress, and belonging.

## Core retention loop

| Loop stage | Product action | Emotional result |
| --- | --- | --- |
| Match recommendation | Suggest a small number of suitable matches | "This is for me." |
| Commitment | Make players, level, payment, and rules clear | "I can trust this." |
| Completion | Confirm attendance and outcome | "My effort counts." |
| Reflection | Give recap and level/reliability update | "I know what happened." |
| Next action | Recommend next match, clinic, squad, or coach | "I know how to continue." |
| Familiarity | Surface known players and home venue rhythm | "I belong here." |

## What creates repeat behavior

| Retention driver | How V1 creates it |
| --- | --- |
| Better match fit | Level confidence, balance feedback, coach validation. |
| Lower coordination burden | Recommended matches, waitlists, reminders, match thread. |
| Social familiarity | Home venue, repeated players, squads. |
| Trust | Reliability score, payment status, cancellation rules. |
| Progress | Recaps, improvement tags, next action. |
| Identity | Match history, level confidence, reliable-player reputation. |
| Venue rhythm | Recurring open-play templates. |

## Weekly habit model

| Weekly moment | Behavior |
| --- | --- |
| Plan | Player checks recommended matches for the week. |
| Commit | Player joins one suitable match. |
| Anticipate | Player sees confirmed players and reminders. |
| Play | Player shows up and completes the match. |
| Reflect | Player gives quick feedback and receives recap. |
| Continue | Player chooses next match, squad, clinic, or coach validation. |

## Emotional retention

Functional retention is "the app is useful."

Emotional retention is "this is becoming part of my life."

| Emotional layer | Product memory |
| --- | --- |
| Welcome | The system knows where I play. |
| Fit | The system knows my level and match preference. |
| Trust | The system remembers I show up. |
| Progress | The system shows how I am changing. |
| Belonging | The system brings me back to familiar people. |
| Guidance | The system helps me choose what to do next. |

## Retention metrics

| Metric | What it proves |
| --- | --- |
| Second-match rate | First experience created enough trust to return. |
| 30-day repeat play | Weekly habit is forming. |
| Match balance rating | Matching quality is working. |
| Same-player repeat rate | Familiarity and squads are forming. |
| Home-venue repeat rate | Venue attachment is forming. |
| Recap completion | Reflection habit is forming. |
| Next-action conversion | Improvement loop is working. |
| No-show rate | Commitment system is working. |

## What weakens retention

| Problem | Why it hurts |
| --- | --- |
| Bad level match | Creates embarrassment, boredom, or distrust. |
| No-shows | Makes open play feel risky. |
| Empty recommendations | Signals no community. |
| Too many irrelevant listings | Makes the user do the system's work. |
| Opaque rating changes | Breaks trust. |
| Recaps without action | Feels like decoration. |
| Generic notifications | Trains users to ignore the app. |

## Retention rule

Every retention mechanic should make the next real-world match better.

If it only increases app usage, it does not belong in V1.
