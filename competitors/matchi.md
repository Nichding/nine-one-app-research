# MATCHi Competitor Report

## Company overview

| Field | Research finding |
| --- | --- |
| Category | Racket-sports booking and venue-management platform |
| Primary sports | Padel, tennis, badminton, table tennis, pickleball, squash, and other racket sports |
| Company | MATCHi AB |
| Positioning | MATCHi says it is a complete racket-sports system and booking platform for venues worldwide. |
| History | MATCHi materials reference more than 10 years of experience; third-party and historical materials indicate the platform dates to around 2012/2013. |

## Target users

| Segment | Needs |
| --- | --- |
| Players | Find venues, book courts, explore memberships and activities, split payments, and manage bookings. |
| Clubs and venues | Online booking, payment handling, memberships, activities, courses, leagues, reduced administration, and profitability tools. |
| Racket-sport communities | A single place to discover facilities and racket-sport programming. |

## Core features

| Area | Features |
| --- | --- |
| Player app | Venue search, available times, favorites, filters by sport/date/time/court type, activities, memberships, booking management, friend invites, and split payments. |
| Venue platform | Booking system for one-time bookings, activities, and courses; payments; administration reduction; league handling and result registration. |
| Sports coverage | Padel, tennis, badminton, table tennis, pickleball, squash, and other racket sports. |
| Terms-covered services | MATCHi terms describe online booking, finding/book/pay services, player matching, and Padelboard tournament tooling. |

## UX flow

1. User registers or logs in.
2. User selects a venue or searches/filter by sport, date, time, and court type.
3. User chooses court, activity, or membership.
4. User selects payment method and confirms booking.
5. User can invite friends, split payment, and manage the reservation.

## Monetization

| Revenue stream | Evidence |
| --- | --- |
| Venue SaaS / agent model | MATCHi markets its venue platform to clubs and asks interested venues to contact them. Public numeric pricing was not found. |
| Booking/payment flow | MATCHi terms say booking price varies by venue and is shown in the app/website at booking time. |
| Player app | Free to download on Google Play/App Store. |
| Tournament/competition tooling | Padelboard by MATCHi is referenced in terms; pricing Unknown. |

## Strengths

| Strength | Why it matters |
| --- | --- |
| Clean booking proposition | MATCHi is straightforward: find a venue, choose time, pay, manage booking. |
| Multi-sport support | Broader than padel-only platforms, useful for mixed racket venues. |
| Venue-focused operations | Reduces admin and supports activities, courses, memberships, and leagues. |
| Payment split | Useful for racket sports where court cost is often divided by players. |

## Weaknesses

| Weakness | Evidence / rationale |
| --- | --- |
| Less visible social/coaching depth than Playtomic | Public pages emphasize booking and venue operations more than social graph, coaching intelligence, or performance analytics. |
| Public pricing opacity | Venue pricing is not publicly exposed. |
| Potential public-match payment edge cases | UK App Store review complains about players taking match slots without paying and blocking others. |
| Review data inconsistency | AppBrain showed no current Google Play rating data despite Google Play listing downloads; app-store signals vary by country. |

## User complaints

| Theme | Reported complaint |
| --- | --- |
| Public match bugs | App Store review says invited/unpaid players can hold slots and prevent others from joining. |
| Multi-court booking | AppBrain surfaced user wishlist for booking multiple courts in one transaction. |
| Calendar and group workflow | AppBrain surfaced requests for calendar subscription and friend groups/chat. |
| Membership edge case | Reddit user reported an issue where one player could not book more than one court at the same time with membership logic. |

## Strategic observations

| Observation | Implication for Nine One |
| --- | --- |
| MATCHi is operationally competent but less emotionally sticky than a social/coaching platform. | Nine One can build identity around player improvement and venue community, not only reservations. |
| Multi-sport venue tooling is table stakes. | Nine One should support padel-first workflows while keeping architecture extendable to tennis/pickleball. |
| Payment slot ownership is a high-friction moment. | Design payment holds, expiration timers, waitlists, and owner controls carefully. |
| Public pricing opacity creates buyer uncertainty. | Transparent venue packages can be a competitive acquisition lever. |

## Opportunity gap for Nine One

| Gap | Product opportunity |
| --- | --- |
| Booking lacks coaching intelligence | Attach match video, AI coaching notes, skill progression, and recommended drills to each booking. |
| Group coordination still spills to WhatsApp | Add squads, recurring groups, trusted substitutes, and one-tap fill-ins. |
| Venue analytics are not player-experience analytics | Connect court utilization to player retention, match quality, repeat bookings, and coach conversion. |
| Level portability across systems is weak | Build verified skill profile and import/manual validation paths. |
