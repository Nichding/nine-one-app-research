# CourtReserve Competitor Report

## Company overview

| Field | Research finding |
| --- | --- |
| Category | Club management and court reservation software |
| Primary sports | Racquet and paddle sports; public pages emphasize pickleball, tennis, and racquet/paddle facilities. |
| Positioning | "All-in-one club management platform" for reservations, communication, payments, and operations. |
| Scale | CourtReserve public homepage cites 2,200+ clubs, 5M+ players, and 25M+ reservations. |
| Orientation | More B2B club-management SaaS than consumer-first marketplace. |

## Target users

| Segment | Needs |
| --- | --- |
| Club owners and operators | Court reservations, payments, memberships, events, lessons, reporting, POS, staff permissions, access control, and integrations. |
| Members and guests | Book courts, register for events, manage profiles, pay, receive alerts, and use club-branded portal/app. |
| Staff and pros | Lesson scheduling, instructor assignments, role permissions, and schedule visibility. |

## Core features

| Area | Features |
| --- | --- |
| Reservations | Court scheduling, booking rules by court/member/time/usage, waitlists, visual calendars, public booking links. |
| Player/member tools | Member portal, profile management, alerts, iOS/Android app, player match-maker. |
| Programming | Events, drop-ins, programs, camps, round robins, clinics, lessons, private/group lessons. |
| Operations | Memberships, staff roles, reporting, payments, invoices/batch billing, POS, pro shop, integrations, access control, branded app on higher plans. |

## UX flow

1. Club configures court inventory, booking rules, member types, pricing, staff, and programs.
2. Member or guest uses the member portal, branded app, or public booking link.
3. User selects court, lesson, event, or program and completes payment/registration.
4. Club staff manage schedules, billing, reporting, members, and operations from the admin system.

## Monetization

| Revenue stream | Evidence |
| --- | --- |
| SaaS subscription | Launch is listed at $199/month monthly or $159/month billed annually; Advance at $369/month monthly or $329/month billed annually; Momentum at $549/month monthly or $499/month billed annually. |
| Enterprise/custom | Custom pricing for more courts, multiple locations, or network memberships. |
| Payments | Integrated payment processing via providers such as Stripe or SafeSave. Specific payment-processing rates were not found in public docs. |
| Add-ons/plan gating | Branded mobile app, branded domains, API access, all integrations, and multi-location capabilities vary by plan. |

## Strengths

| Strength | Why it matters |
| --- | --- |
| Deep club operations | Covers more back-office workflows than consumer-first booking apps. |
| Transparent public pricing | Helps smaller clubs understand cost before sales contact. |
| Branded member experience | Supports custom-branded portals and mobile apps. |
| Public booking | Allows non-members to book via shareable link without requiring a CourtReserve account. |
| Mature admin rule system | Booking rules, waitlists, memberships, staff roles, and billing fit complex clubs. |

## Weaknesses

| Weakness | Evidence / rationale |
| --- | --- |
| Less marketplace discovery | It is club-centered, not primarily a cross-club player marketplace. |
| Less padel-specific social identity | Public materials do not emphasize padel-style open matches, level ladders, or social opponent discovery as strongly as Playtomic. |
| Mobile UX complaints | App Store reviews mention calendar and usability frustration. |
| Higher operating complexity | Inference: feature-rich admin tools may require setup/training and can feel heavy for small padel venues. |

## User complaints

| Theme | Reported complaint |
| --- | --- |
| Calendar UX | App Store review snippet says "My complaint is the calendar." |
| App reliability/usability | App Store snippets describe serious issues and frustration. |
| Mobile optimization | Reddit discussion about court-booking apps says platforms including CourtReserve can have too many menus and weak mobile optimization; anecdotal, not independently verified. |

## Strategic observations

| Observation | Implication for Nine One |
| --- | --- |
| CourtReserve owns the operations-heavy end of the market. | Nine One should avoid building a bloated admin suite before proving padel-specific workflows. |
| Public booking without account is a useful acquisition mechanism. | Nine One should support friction-light guest booking while nudging players into richer profiles after value is clear. |
| Club control is a major differentiator. | Nine One can combine club control with player-facing intelligence and community. |
| Transparent pricing is a strength. | Nine One should consider simple packages for early venues. |

## Opportunity gap for Nine One

| Gap | Product opportunity |
| --- | --- |
| Club software is operational, not coaching-led | Build venue management around player development, smart coaching, and recurring engagement. |
| Padel match formation needs stronger identity | Add open matches, skill confidence, attendance reputation, squads, and coach validation. |
| Complex admin can overwhelm smaller venues | Offer padel-ready defaults: peak/off-peak pricing, court rules, open-play templates, clinics, and league formats. |
| Weak consumer marketplace | Build localized discovery and social liquidity around Nine One partner venues. |
